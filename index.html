Лекция 1. Архитектура будущего приложения

Сегодня мы не пишем код — мы строим план. Это важно, чтобы дальше не «чинить» структуру приложения по ходу.

1) Минимальная архитектура

Для старта мы используем современный подход Apple — SwiftUI для UI и Model для данных. Вы пока не знаете, что такое struct и class, поэтому сегодня объясним именно их роль в проекте.

Model (Данные)

Описание книги: название, автор, год, статус, обложка.

View (Экран)

Список книг, поиск, фильтр по статусу.

Storage (Хранилище)

Локальное сохранение списка на Mac (без облака).

2) Почему мы начинаем со struct

В Swift есть два основных способа описывать данные: struct и class. Мы начнем со struct, потому что:

  • Он проще в обучении и лучше подходит для «простых данных» вроде книги.
  • SwiftUI оптимизирован под использование struct.
  • Для ваших задач это более безопасный и предсказуемый вариант.

3) Паттерны и принципы (без перегруза)

Термин «паттерн» — это просто устойчивое решение типичной задачи. Мы не будем заучивать теорию, а введем минимально нужные элементы:

  • Разделение ответственности: данные отдельно от UI.
  • Однонаправленный поток данных: UI читает модель, а не наоборот.
  • Маленькие шаги: каждую лекцию добавляем один слой.

На следующей лекции мы создадим первый struct Book и отобразим тестовый список в интерфейсе. Подробно разберем, как SwiftUI строит экран.

Лекция 2. Первый экран и модель данных

На этом шаге мы впервые пишем код. Цель — создать модель книги и показать тестовый список на экране. Это даст базовое понимание struct и того, как SwiftUI строит интерфейс.

1) Что такое struct (на практике)

struct — это «контейнер» для данных. Мы создаем тип Book с набором свойств: название, автор, год, статус, обложка. Именно так мы описываем «книгу» для приложения.

Минимальная модель Book

Название, автор, год, статус, обложка — это наши свойства.

struct Book { ... }

2) Как SwiftUI строит экран

SwiftUI работает декларативно: мы «описываем», как должен выглядеть экран, а система сама обновляет его при изменениях данных. Для списка используем List, а для строки книги — простой HStack.

3) Практика в Xcode

  1. Откройте проект BookTracker.
  2. Создайте новый файл Swift File и назовите его Book.
  3. Опишите struct Book с нужными свойствами.
  4. В ContentView добавьте тестовый массив книг.
  5. Отобразите список в List и выведите название и автора.

Результат лекции 2

  • Создан struct Book и вы понимаете роль его свойств.
  • Отображен тестовый список книг в SwiftUI.
  • Вы увидели, как работает декларативный UI и почему это удобно.
